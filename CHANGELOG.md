# Changelog — ветка `babylon-ts`

Переписывание 3D-шутера с однофайлового Three.js (ветка `main`) на
TypeScript + Babylon.js + Vite. Формат: новое сверху.

---

## BSP: текстуры интерьера ангара (текущая)

### Добавлено
- **Асфальтовый пол входа**: полы внутри ангара/туннеля (`tnnl_flr*`, `labflr`,
  `duct_flr`, `crete4_flr`, `*_flr\d`) → новая категория `asphalt` (тёмно-серый
  спекл, как мост снаружи).
- **Бетон**: остальные стены/поверхности интерьера → `concrete`.
- **Вентиляция**: воздуховоды/решётки (`duct`, `vnt`/`vent`, `pfab`, `galv`,
  `grille`) → категория `vent` — процедурная текстура металла с горизонтальными
  жалюзи.
- **Кабинет/консоли**: `comp*`, `generic*`, `viewscreen`, `dsk`, `babtech`,
  `fifties`, `silo`, двери-`dr` → категория `office` — панельная сетка с тёмными
  экранами и зелёными индикаторами.

### Исправлено
- Наружный двор (`out_pave2`) и крыша (`out_gravel`) исключены из `asphalt`,
  чтобы улица не потемнела — асфальт только внутри.

## BSP: текстуры ящика/грузовика, колёса, 3 лестницы

### Добавлено
- **Текстура ящика**: отдельная категория `crate` (была общей с `wood`) — процедурная
  текстура с горизонтальными досками и диагональными металлическими рейками-крепления.
- **Текстура и колёса грузовика**: части `trk_*` теперь получили материалы по смыслу —
  кузов/дверь/борт/капот/тент → `vehicle` (оливково-хаки), обод → `rim` (серебристый
  металл), шина → `tire` (чёрная резина). У модели грузовика в мире BSP нет реальной
  геометрии шин (только текстуры без граней) — добавлены 4 колеса-цилиндра по углам
  кузова (`trk_bed`), касаются земли — грузовик больше не «висит в воздухе».
- **Три лестницы** (крутые пандусы, `metadata.floor`, как и остальная вертикальная
  физика карты) для подъёма на крыши:
  1. У контейнеров/ящика рядом с ангаром (реальный проход в стену найден на x=-30 —
     проверено практическим проходом, т.к. соседние точки блокировались настоящей стеной).
  2. С другой стороны того же комплекса, в верхнем правом углу миникарты.
  3. У здания рядом с грузовиком, в углу.
  Все три подтверждены подъёмом с земли (eval): 1.7→9.7, 1.7→12.7, 1.7→10.6 (высоты целей
  8/11/9 — небольшой перелёт, ожидаемо для одометрии теста, не для геймплея).

### Не сделано (честно, по ограничению времени)
- **Реальный проём (дверь)** в стене BSP около второй лестницы — не вырезан. Вырезать
  проход в уже собранной BSP-геометрии по отдельной грани — отдельная по объёму задача
  (не совпадает с нашей текущей категоризацией по группам текстур). Лестница у этого места
  добавлена и работает, просто входа с земли там по-прежнему нет, только подъём.

### Проверено (preview, eval)
- Материалы: `bsp_trk_*` → `proc_vehicle`/`proc_rim` корректно; `proc_crate` создан.
- 4 колеса на нужной высоте (y=0.55, касаются земли) по углам кузова.
- Все 3 лестницы: подъём с 1.7 до целевой высоты подтверждён покадровой трассировкой.

## BSP: полосы на текстурах + непроходимость

### Исправлено
- **Полосы/муар на текстурах.** Процедурные `DynamicTexture` (кирпич/плитка/крап) создавались
  с `generateMipMaps=false` — без мипмапов тайловые текстуры пола дают резкий муар под
  углом обзора. Включены мипмапы + `anisotropicFilteringLevel=8` на все материалы BSP
  (и реальные WAD, и процедурные).
- **Угол взгляда на спавне.** `info_player_start` в BSP хранит `angle` (куда смотреть) —
  раньше игнорировался, камера всегда стартовала лицом на «север» карты независимо от
  замысла автора. Теперь угол читается из entity-лампы и конвертируется в поворот камеры.
- **Игрок упирался во всё «невидимое» при ходьбе вперёд — три независимые причины,
  найдены экспериментально (диагностика по вершинам геометрии рядом с точкой
  застревания, т.к. `pickWithRay` не всегда совпадает с реальной системой коллизий):**
  1. Декоративный **грузовик** (кузов/дверь/тент — `trk_*`) и настенные **декор-панели**
     (`~lab1_comp7` и т.п.) — тонкие детали с вогнутыми стыками, в которых эллипсоид
     игрока залипал. Явный список декоративных текстур — без коллизии.
  2. **Пол** (`out_pave2` и подобные): коллизия Babylon по горизонтальным граням на стыках
     соседних плит иногда ложно ведёт себя как стена. Грани теперь классифицируются по
     нормали при разборе BSP: почти горизонтальные (|normal.y| > 0.7) — без коллизии
     Babylon, только `metadata.floor` для собственной вертикальной физики (как на
     рукописных картах).
  3. **Низкие бордюры/поребрики** (`out_wall8` и т.п., высота < 0.6 — колено): по имени
     похожи на стену, но Babylon блокирует ими всей высотой эллипсоида, хотя реальный
     игрок просто перешагнул бы. Решение принимается **по каждой отдельной грани** (не по
     агрегированной группе текстуры — та же текстура может быть и низким бордюром, и
     высокой стеной в разных местах карты): низкие — без коллизии + `metadata.floor`
     (перешагиваются вертикальной физикой), капитальные категории (стены/полы/ящики/
     контейнеры/здания) остаются твёрдыми независимо от габаритов.

### Проверено (preview, eval)
- Прогон вперёд от спавна: было 5.3 → 6.5 → 9.1 юнита (последовательные фиксы) → **71.5
  юнитов** после всех трёх причин, упёрлись в настоящую стену здания `c_bldg2_2`
  (реальная архитектура, ожидаемо). Проверено ещё 4 направления/точки — везде разумная
  дистанция, без ложных застреваний.
- 180 групп граней (было 103): 107 помечены `metadata.floor` (пол/потолок/низкие бордюры),
  52 капитальные стены с коллизией; рендер 3.61 мс/кадр (~277 fps).

## BSP: текстуры и миникарта

### Добавлено
- **Парсер WAD3** (`src/wad.ts`): читает директорию `cs_assault.wad`, декодирует miptex
  (палитра 256×RGB + индексы) в RGBA-пиксели.
- **Реальные текстуры где есть WAD-совпадение** (18 из 22 текстур WAD используются в
  world-геометрии): грани группируются по имени текстуры (было — один серый меш на всю
  карту), для каждой группы считается UV из `texinfo` (векторы S/T + offset, делённые на
  ширину/высоту текстуры из BSP — то же значение, что использовал автор карты). Группы с
  WAD-совпадением получают `RawTexture` из декодированных пикселей.
- **Процедурные текстуры по категориям** для остальных ~85 групп (нет WAD с оригиналом):
  эвристика по имени (`concrete`/`brick`/`metal`/`wood`/`floor`/`glass`/`light`/`grass`/
  `generic`) → одна из существующих процедурных генераций (кирпич/плитка/крап), материал
  переиспользуется на все группы одной категории.
- **Миникарта BSP**: топ-вид запекается **один раз** при загрузке — все грани мира
  проецируются на XZ и заливаются на canvas 256×256 (силуэт геометрии), а не рисуются
  каждый кадр. `drawMinimap()` в BSP-режиме просто накладывает этот фон + стрелку игрока
  (те же center/span, что при запекании, — вынесены в mutable `mmCenterX/mmCenterZ/mmSpan`).

### Проверено (preview, eval)
- 103 группы граней: 18 с реальной WAD-текстурой, 85 процедурных по категориям; скриншот
  подтверждает кирпичный фасад (реальная текстура) и плитку тротуара.
- **Миникарта — тройная перепроверка** (как для процедурных карт): (1) маппинг точен —
  центр bounds → канва [84,84], угол (minX,maxZ) → верх-лево, (maxX,minZ) → низ-право;
  (2) стрелка: взгляд +z → вверх (−1.57), +x → вправо (0); (3) стрелка реально движется —
  из угла карты в противоположный угол канвы при переходе игрока между углами мира.
- Движение/коллизия по мультимеш-геометрии не пострадали; рендер 1.44 мс/кадр (~700 fps)
  несмотря на 103 отдельных меша.

### Исправлено

### Проверено (preview, eval)

## Система карт + вторая карта

### Добавлено
- **Переключение карт (клавиша M).** Уровень вынесен в систему карт: меши текущей карты
  собираются в «приёмник» (`sink`), `loadMap(i)` выгружает прошлую (dispose мешей/дверей/
  целей/кубиков), очищает `targets/pickups/doors/footprints`, сбрасывает счётчики, строит
  новую, ставит спавн и пересчитывает мировые bbox. Реестр карт + тост с названием.
  Отложенные респавны старой карты гасятся счётчиком поколений (`mapGen`).
- **Карта 1 «Арена (город)»** — прежний уровень, теперь как пункт реестра (сохранён).
- **Карта 2 «cs_assault» (клон)** по плану: склад 40×28 (пол y=0, стены 7), 2-й ярус —
  галерея по трём стенам (y=3.5, T-зона) с перилами и пандусом; 4 входа — **A** ворота (юг),
  **B** окно через фургон (заезд на крышу y≈2.2 → мостик в окно → галерея), **C** боковая
  дверь (восток), **D** задняя дверь (север); улица с CT-спавном, ящики-укрытия, мишени на
  галерее и полу. Двери A/C/D — авто-открытие.
- Мишени/кубики получили высоту спавна (`baseY`) — для целей на галерее; респавн учитывает высоту.

### Проверено (preview, eval)
- M циклит карты (Город → cs_assault → Город), тост показывается; выгрузка чистая
  (0 «утёкших» мешей при смене).
- Арена: 8 целей / 5 следов; cs_assault: склад 13 стен / 3 двери / 3 секции галереи.
- На cs_assault: вход через ворота A (спавн → внутрь склада), подъём на галерею по пандусу
  (y 1.7→5.3), цель на галерее простреливается (bbox корректен), путь B (фургон→окно→галерея)
  по высотам проходит, дверь C авто-открывается.

## Геймплей-правки по фидбэку

### Исправлено
- **Лестница теперь проходима.** Ступени были развёрнуты высокой стороной к спавну
  (упираешься как в стену) и каждая блокировала горизонтально. Теперь ступени —
  визуальные (`checkCollisions=false`), высота возрастает к платформе; подъём
  обеспечивает невидимый наклонный **пандус** (поверхность ходьбы через `metadata.floor`,
  который видит луч опоры). Платформа тоже сделана floor-поверхностью без боковой
  коллизии — заход с лестницы без «порога». Проверено: высота опоры растёт 0→3.6 по
  лестнице; игрок поднимается с y=1.7 на платформу и проходит дальше без блока.
- **Двери открываются автоматически** при подходе (< 3 м) и закрываются при отдалении —
  больше не нужно жать E (это было неочевидно, игрок не мог зайти). Закрытая дверь
  блокирует, открытая — нет. Проверено: подошёл → −90° и коллизия off; отошёл → 0° и коллизия on.
- **Убраны невидимые препятствия:** крыши зданий были коллизионными (невидимый потолок) —
  отключена коллизия. От спавна вперёд теперь чистый проход (дошёл z=−26 → 31 без упора).

### Текстуры
- Кирпич мельче и реалистичнее (uScale 2 → 5), плитка пола крупнее и контрастнее
  (uScale 16 → 10, светлее плита / темнее шов).

## Правки + миникарта

### Добавлено
- **Миникарта** (вид сверху, справа вверху): контур границы, силуэты зданий, мишени
  (красные точки), кубики (бирюзовые), игрок жёлтой стрелкой по направлению взгляда.
  Маппинг мир→карта: `x` вправо, `z` вверх; стрелка строится из мировой точки впереди
  игрока тем же маппингом, что и позиции (направление гарантированно совпадает).
- **Текстура пола**: плиточный `DynamicTexture` (сетка плит с тёмными швами + крап) вместо
  едва заметного крапа — пол теперь чётко читается.

### Исправлено
- **Движение на любой раскладке (WASD == ЦФЫВ).** Камера использовала ввод по `keyCode`;
  заменено на свой по **`event.code`** (`KeyW/A/S/D` + стрелки) с движением через
  `cameraDirection` (коллизии сохранены). Клавиши действий тоже переведены на `event.code`
  (`KeyR`, `KeyE`, `Digit1/2`, `Space`). При потере фокуса окна нажатия сбрасываются.
- **Невидимые стены.** Материалы построек имели `backFaceCulling=true` по умолчанию, поэтому
  изнутри здания стены не рисовались, хотя коллизия оставалась. Отключён backface culling
  на стенах/бетоне/крышах/дверях — стены видны с обеих сторон.

### Проверено (preview, eval)
- Раскладка: событие `code:'KeyW', key:'ц'` кладёт `KeyW` в набор нажатий; keyup убирает.
- Стены: `backFaceCulling=false` на brick/brick2/concrete/roof/door. Пол: текстура `floor`.
- **Миникарта — тройная перепроверка:** (1) маппинг точен (центр→[84,84], +x→право,
  +z→верх, −x→лево); (2) стрелка: взгляд +z → вверх (−1.57), +x → вправо (0);
  (3) объекты обновляются — после килла 7 красных точек, после подбора 5 бирюзовых,
  стрелка игрока на месте.

## Фаза 4 — звук и полировка

### Добавлено
- **Звуки через WebAudio** (синтез на лету, без файлов): осциллятор + шумовой буфер с
  ФНЧ. Выстрел различается для пистолета/SMG, отдельные звуки попадания, килла (нисходящий
  тон) и перезарядки (двойной клик). `AudioContext` создаётся лениво и резюмируется по
  клику в оверлее (требование браузеров — звук только после жеста).
- **Процедурные текстуры** (`DynamicTexture`, рисуются на canvas): кирпичная кладка двух
  расцветок для стен (со случайной вариацией яркости кирпичей), крапчатый бетон для пола
  (uScale 24), платформы/ступеней и границы. Цвет материала сброшен в белый, чтобы не
  затемнять текстуру.
- **Покачивание оружия** при ходьбе (определяется по смещению камеры за кадр) и плавная
  **просадка ствола** при перезарядке — оружие уходит вниз и наклоняется.

### Проверено (preview, eval)
- Текстуры назначены (brick uScale 2, ground 24, concrete); консоль без ошибок.
- `fire()` со звуком не падает (`AudioContext` в headless создаётся без исключений),
  килл засчитывается. Реальный рендер 1.28 мс/кадр (~780 fps) — Фаза 4 не просадила перф.
  (1 fps в preview — троттлинг фоновой вкладки `document.hidden`, не игра.)

## Фаза 3 — мир и взаимодействие

### Добавлено
- **Открываемые двери** (клавиша `E` / `У`): у каждого здания дверь на петле

## Фаза 2 — оружейная система

### Добавлено
- **Две единицы оружия** через интерфейс `Weapon` (имя, узел, вспышка, магазин,
  патроны, темп `interval`, авто/полуавто, урон в тело/голову, отдача, время перезарядки):
  - **Пистолет** — детальная модель (затвор, рама, рукоять под углом, мушка, ствол-цилиндр),
    магазин 12, полуавтомат (170 мс), урон 50 / 100 (хедшот), перезарядка 800 мс.
  - **SMG** — модель (корпус, верхняя планка, наклонный магазин, рукоять, приклад),
    магазин 30, **автоогонь** при удержании ЛКМ (75 мс), урон 24 / 55, перезарядка 1100 мс.
- **Переключение оружия** клавишами `1` / `2` (`switchWeapon`) — модели через `setEnabled`,
  HUD показывает имя и патроны активного ствола.
- **Дульная вспышка** — билборд-диск с эмиссивным материалом у дульного среза каждого
  ствола, вспыхивает на ~45 мс со случайным масштабом.
- **Hit-marker** — символ `✕` в центре экрана при попадании, краснеет на хедшоте (DOM).
- **Всплывающие цифры урона** — число у точки попадания (через `Vector3.Project`),
  всплывает и тает за 0.6 с; на хедшоте — со звездой `★` и крупнее (DOM).
- **Авто-перезарядка** при опустошении магазина; ручная — `R` (учитывает раскладку «К»).
- Материалы оружия: воронёная сталь (`bluedMat`) и полимер (`polyMat`, `magMat`).
- У каждой мишени теперь своя копия материала — подсветка попадания не задевает остальные.
- Отладочный хук `window.GAME` расширен (`weapons`, `fire`, `switchWeapon`, `getCur`).

### Исправлено
- **Критический баг прицеливания:** Babylon обновляет мировой bounding box лениво.
  У статичных мешей (стены, платформа, ступени, мишени), которым `position` задавали
  уже после создания, мировой bbox оставался в начале координат `(0,0,0)`.
  Из-за этого `pickWithRay` промахивался — **попасть нельзя было ни по одной цели**
  (патрон тратился, вспышка была, но урона ноль). Лечится принудительным
  `refreshBoundingInfo()` + `computeWorldMatrix(true)` по всем мешам после сборки сцены.
  Заодно чинит опору на платформе/ступенях (луч вниз тоже пикает по bbox).

### Исправлено
- **Чёрный экран:** `camera.moveWithCollisions(...)` — такого метода у камеры нет
  (это метод `AbstractMesh`). Он бросал исключение в `onBeforeRenderObservable` каждый
  кадр, поэтому ни один кадр не отрисовывался (исключение глоталось внутри `requestAnimationFrame`,
  и консоль выглядела чистой). Вертикаль переписана на ручное движение `position.y`
  с лучом вниз для поиска опоры; горизонтальные коллизии остались на встроенном вводе камеры.
- **Пересвет:** интенсивности света снижены (hemi 0.8 / sun 0.85), земля затемнена —
  раньше пол выбивался в белый.
