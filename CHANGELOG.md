# Changelog — ветка `babylon-ts`

Переписывание 3D-шутера с однофайлового Three.js (ветка `main`) на
TypeScript + Babylon.js + Vite. Формат: новое сверху.

---

## Фаза 2 — оружейная система (текущая)

### Добавлено
- **Две единицы оружия** через интерфейс `Weapon` (имя, узел, вспышка, магазин,
  патроны, темп `interval`, авто/полуавто, урон в тело/голову, отдача, время перезарядки):
  - **Пистолет** — детальная модель (затвор, рама, рукоять под углом, мушка, ствол-цилиндр),
    магазин 12, полуавтомат (170 мс), урон 50 / 100 (хедшот), перезарядка 800 мс.
  - **SMG** — модель (корпус, верхняя планка, наклонный магазин, рукоять, приклад),
    магазин 30, **автоогонь** при удержании ЛКМ (75 мс), урон 24 / 55, перезарядка 1100 мс.
- **Переключение оружия** клавишами `1` / `2` (`switchWeapon`) — модели через `setEnabled`,
  HUD показывает имя и патроны активного ствола.
- **Дульная вспышка** — билборд-диск с эмиссивным материалом у дульного среза каждого
  ствола, вспыхивает на ~45 мс со случайным масштабом.
- **Hit-marker** — символ `✕` в центре экрана при попадании, краснеет на хедшоте (DOM).
- **Всплывающие цифры урона** — число у точки попадания (через `Vector3.Project`),
  всплывает и тает за 0.6 с; на хедшоте — со звездой `★` и крупнее (DOM).
- **Авто-перезарядка** при опустошении магазина; ручная — `R` (учитывает раскладку «К»).
- Материалы оружия: воронёная сталь (`bluedMat`) и полимер (`polyMat`, `magMat`).
- У каждой мишени теперь своя копия материала — подсветка попадания не задевает остальные.
- Отладочный хук `window.GAME` расширен (`weapons`, `fire`, `switchWeapon`, `getCur`).

### Исправлено
- **Критический баг прицеливания:** Babylon обновляет мировой bounding box лениво.
  У статичных мешей (стены, платформа, ступени, мишени), которым `position` задавали
  уже после создания, мировой bbox оставался в начале координат `(0,0,0)`.
  Из-за этого `pickWithRay` промахивался — **попасть нельзя было ни по одной цели**
  (патрон тратился, вспышка была, но урона ноль). Лечится принудительным
  `refreshBoundingInfo()` + `computeWorldMatrix(true)` по всем мешам после сборки сцены.
  Заодно чинит опору на платформе/ступенях (луч вниз тоже пикает по bbox).

## Фаза 1 — играбельное ядро

### Добавлено (каркас)
- Проект Vite + `@babylonjs/core` + TypeScript (strict): `package.json`, `tsconfig.json`,
  `vite.config.ts` (`base: './'` для GitHub Pages), `.gitignore`, `index.html`, `src/main.ts`.
- **Дневная сцена:** небо `#9ec9f0`, линейный туман 90–320, HemisphericLight + DirectionalLight.
- **FPS-камера** (`UniversalCamera`): свой mouse-look на pointer-lock, WASD, эллипсоидные
  коллизии, ручная гравитация и прыжок (Пробел).
- **Карта:** земля 220×220, 5 зданий (стены с дверным проёмом + крыша, дверь без коллизии),
  бетонная граница, платформа + лестница из 7 ступеней (перепад высоты).
- **Мишени:** 8 красных боксов с HP.
- **Стрельба:** хитскан `getForwardRay` + `pickWithRay`, хедшоты, отдача, DOM-HUD
  (патроны / убито / прицел), оверлей «кликни, чтобы играть».

### Исправлено
- **Чёрный экран:** `camera.moveWithCollisions(...)` — такого метода у камеры нет
  (это метод `AbstractMesh`). Он бросал исключение в `onBeforeRenderObservable` каждый
  кадр, поэтому ни один кадр не отрисовывался (исключение глоталось внутри `requestAnimationFrame`,
  и консоль выглядела чистой). Вертикаль переписана на ручное движение `position.y`
  с лучом вниз для поиска опоры; горизонтальные коллизии остались на встроенном вводе камеры.
- **Пересвет:** интенсивности света снижены (hemi 0.8 / sun 0.85), земля затемнена —
  раньше пол выбивался в белый.

### Проверено (preview)
- Сцена рендерится при 60 FPS, `scene.isReady()` = true, материалы скомпилированы.
- Хитскан-луч попадает в мишень; переключение оружия, урон, килл, hit-marker и
  цифры урона подтверждены через `window.GAME` + eval.
