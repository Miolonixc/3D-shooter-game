# 3D Shooter → TypeScript + Babylon.js — план переписывания

Ветка: **`babylon-ts`** (main с рабочей vanilla-версией не трогаем, пока новая не догонит).
Цель: тот же «играть по ссылке» (GitHub Pages), но на TS + Babylon.js — типобезопасно,
модульно, со встроенными коллизиями/гравитацией/камерой.

## Стек и сборка
- **TypeScript** + **Vite** (`npm run dev` / `build`), **@babylonjs/core**.
- `base: './'` в vite.config — ассеты на под-пути Pages (`/3D-shooter-game/`).
- Деплой: GitHub Actions собирает `dist/` и публикует в Pages (Pages source → «GitHub Actions»).

- Текстура пола (плитка). Движение на любой раскладке (WASD == ЦФЫВ, по `event.code`).

### ✅ Сделано (каркас + играбельное ядро)

- `src/main.ts`: дневная сцена, FPS-камера (свой mouse-look + WASD + коллизии), ручная

  платформа+лестница (перепад высоты), мишени, пистолет (привязан к камере), хитскан-стрельба
  с хедшотами, перезарядка, отдача, HUD (DOM: патроны/убито/прицел).
- `npm run build` проходит (бандл ~5 МБ, gzip ~1.1 МБ).

### ✅ Проверено в браузере (preview)
- Сцена рендерится (небо, здания, платформа, оружие, HUD), 60 FPS, `scene.isReady()`.
- Хитскан-стрельба: луч из камеры попадает в мишень (проверено через `pickWithRay`).
- **Исправлен баг чёрного экрана:** `camera.moveWithCollisions` не существует (это метод
  AbstractMesh, не камеры) — выбрасывал исключение в render-loop каждый кадр, кадр не
  рисовался. Вертикаль переписана на ручное движение `position.y` + луч вниз для поиска опоры.

### Фаза 2 — оружие (паритет с vanilla + лучше) ✅
- [x] Детальная модель пистолета (затвор/рама/рукоять/мушка/ствол), воронёный + полимерный материал.
- [x] **SMG** (корпус/планка/магазин/рукоять/приклад) + переключение **1/2**, автоогонь
  (зажать ЛКМ), у каждого ствола свой магазин/урон/темп/отдача/перезарядка.
- [x] Дульная вспышка (билборд), hit-marker (✕, краснеет на хедшоте), всплывающие цифры урона
  (★ + число на хедшоте), авто-перезарядка при опустошении магазина.
- [x] **Багфикс:** Babylon лениво обновляет мировой bounding box — у статичных мешей с заданной
  после создания позицией он оставался в (0,0,0), и стрельба промахивалась по всем целям.
  Принудительный `refreshBoundingInfo()` + `computeWorldMatrix(true)` после сборки сцены.

## Решения / открытые вопросы
- Бандл сейчас 5 МБ из-за `import * as B from '@babylonjs/core'`. Ок для v1; ужать в Фазе 5.
- Деплой: пока проверяем локально (preview). Перед «боевым» включением — workflow + переключить
  Pages на сборку из Actions (разовая настройка в репозитории).
- Когда `babylon-ts` догонит/перегонит vanilla — смержить в `main` (тогда Pages отдаёт новую версию).
